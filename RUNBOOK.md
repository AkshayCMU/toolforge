## 3. Required External Data

The pipeline needs raw ToolBench JSON files. These are **not bundled** in this repo
because the full dataset is several GB.

### Option A — Full ToolBench (recommended for diversity experiment)

The full dataset should be present at a **sibling directory** of this repo:

```
SAP/
├── toolbench_raw/
│   └── data/
│       └── data/
│           └── toolenv/
│               └── tools/
│                   ├── Advertising/
│                   ├── Business/
│                   ├── ... (50 categories)
│                   └── Travel/
└── toolforge/    ← you are here
```

If `../toolbench_raw/` exists with that layout, `toolforge build` will find it
automatically (no flags needed).

**To verify it's there:**
```bash
ls ../toolbench_raw/data/data/toolenv/tools/
# Should list ~50 category folders
```

### Option B — Mini Fixture (bundled, always works)

The `tests/fixtures/toolbench_mini/` directory is included in this repo. It contains
a representative sample of the same 8 categories used in the full build. Use it when:
- The full ToolBench data is not available
- You want a fast smoke test (~30 seconds cold, 0 seconds warm)

**To build with the mini fixture:**
```bash
toolforge build \
  --data-dir tests/fixtures/toolbench_mini/data/toolenv/tools \
  --examples-dir tests/fixtures/toolbench_mini/data/toolenv/tools
```

> **Note:** Mini-fixture builds produce a smaller registry (fewer endpoints). The
> diversity experiment and e2e test work best with the full ToolBench data.

## 6. Running the Pipeline

### 6.1 Build (MUST run first)

Normalizes ToolBench data, runs semantic typing (LLM), builds the knowledge graph.
Outputs all files into `artifacts/`. **Nothing downstream works without this step.**

```bash
# With full ToolBench data (auto-detects ../toolbench_raw/):
toolforge build

# With explicit paths:
toolforge build \
  --data-dir ../toolbench_raw/data/data/toolenv/tools \
  --examples-dir ../toolbench_raw/data/data/toolenv/tools

# With mini fixture:
toolforge build \
  --data-dir tests/fixtures/toolbench_mini/data/toolenv/tools \
  --examples-dir tests/fixtures/toolbench_mini/data/toolenv/tools
```

**Expected output** (cold run, full data):
```
Building registry...  ✓  (500 endpoints, 60 tools, 8 categories)
Semantic typing...    ✓  (~500 LLM calls, ~8-12 minutes)
Building graph...     ✓  (105 CHAINS_TO edges)
Artifacts written to artifacts/
```

**Second run:** 0 LLM calls, ~10 seconds. The semantic typing pass is fully cached.

**Verify the build succeeded:**
```bash
python -c "
import json
reg = json.load(open('artifacts/registry.json'))
print(f'Registry: {len(reg)} tools')
tot = sum(len(t[\"endpoints\"]) for t in reg)
print(f'Endpoints: {tot}')
"
# Should print: Registry: 60 tools, Endpoints: ~500
```

### 6.2 Generate Conversations

```bash
# Quick smoke test (2 conversations)
toolforge generate --n 2 --seed 42

# Larger sample (requires confirmation for n > 10)
toolforge generate --n 20 --seed 42 --out runs/sample.jsonl
```

Output is a JSONL file. Each line is one conversation record with messages,
tool calls, tool outputs, judge scores, validation results, and metadata.

### 6.3 Evaluate a Run

```bash
toolforge evaluate \
  --in runs/sample.jsonl \
  --diversity \
  --out reports/sample.json
```

Writes two files: `reports/sample.json` (machine-readable) and `reports/sample.md`
(human-readable summary with all metrics).

### 6.4 Compare Two Runs

```bash
toolforge compare \
  --a reports/run_a.json \
  --b reports/run_b.json \
  --out reports/comparison.md
cat reports/comparison.md
```

## 7. Running the Diversity Experiment

> **Cost warning:** ~4,000 LLM calls total, estimated ~$12. Confirm before running.
> On a warm cache (if runs were done on the submitter's machine), this is near-zero calls.

The experiment compares two generation modes:
- **Run A** (`--no-cross-conversation-steering`): baseline, uniform chain sampling
- **Run B** (default): inverse-frequency steering + hard rejection caps

```bash
# Step 1: Build (if not already done)
toolforge build

# Step 2: Run A — steering OFF (baseline)
toolforge generate \
  --n 120 --seed 42 \
  --no-cross-conversation-steering \
  --out runs/run_a.jsonl
# Confirm the n>10 prompt with: y

# Step 3: Run B — steering ON
toolforge generate \
  --n 120 --seed 42 \
  --out runs/run_b.jsonl
# Confirm with: y

# Step 4: Evaluate both
toolforge evaluate --in runs/run_a.jsonl --diversity --out reports/run_a.json
toolforge evaluate --in runs/run_b.jsonl --diversity --out reports/run_b.json

# Step 5: Compare
toolforge compare --a reports/run_a.json --b reports/run_b.json --out reports/comparison.md
cat reports/comparison.md
```

**Metrics compared:**
- Tool coverage entropy (higher = more uniform tool usage)
- Distinct tool bigrams / distinct-2 (higher = more varied chains)
- Task embedding dispersion (higher = more semantically diverse tasks)
- Mean judge score (expected to be similar between runs)

## 10. Cost & Call-Count Reference

| Operation | LLM calls (cold) | LLM calls (warm) | Approximate cost |
|-----------|-----------------|-----------------|-----------------|
| `toolforge build` (full data) | ~500 | 0 | ~$0.50 |
| `toolforge build` (mini fixture) | ~50 | 0 | ~$0.05 |
| `toolforge generate --n 10` | ~150 | 0 | ~$0.15 |
| `toolforge generate --n 100` | ~1,500 | 0 | ~$1.50 |
| `toolforge generate --n 120` | ~1,800 | 0 | ~$1.80 |
| Diversity experiment (Run A + B, n=120 each) | ~3,600 | 0 | ~$4–8 |
| E2E test (`--n 100`) | ~1,500 | ~0 | ~$1.50 |

**Warm cache:** Once a run has been made, the `.cache/llm/` directory stores every
response keyed by `(model, prompt_hash, schema_hash)`. Re-running with the same data
and same prompts hits the cache every time — zero API calls, same output.

> The submitter's machine has a warm cache from prior runs. If you are reproducing
> from a fresh clone without that cache, budget for cold-run costs above.

## 11. Troubleshooting

### `registry.json` is empty (`[]`) after build

The build ran but found no tools. Most likely cause: wrong `--data-dir` path.

```bash
# Verify the data path is correct
ls ../toolbench_raw/data/data/toolenv/tools/ | head -5
# Should list category names (Advertising, Business, ...)

# Re-run with explicit path
toolforge build \
  --data-dir ../toolbench_raw/data/data/toolenv/tools \
  --examples-dir ../toolbench_raw/data/data/toolenv/tools
```

### `ANTHROPIC_API_KEY` errors

```bash
# Check .env exists and has the key
cat .env | grep ANTHROPIC
# Should print: ANTHROPIC_API_KEY=sk-ant-...  # nocheck

# If not: copy the template and fill it in
cp .env.example .env
# Edit .env with your key
```

### `toolforge` command not found

```bash
# Make sure venv is activated and package is installed
source .venv/bin/activate
pip install -e .
which toolforge   # should resolve to .venv/bin/toolforge
```

### `generate --n 100` times out or hangs

The generate command makes ~15 LLM calls per conversation. For n=100, that's ~1,500
calls. If cold, expect 15–30 minutes. Check that:
1. Your API key has sufficient credits / rate limits
2. The `.cache/llm/` directory is being written (check its file count grows)

### Python version issues

`mem0ai` has wheel gaps on some Python 3.12 builds. If you hit source-build failures:
```bash
python --version  # confirm 3.11.x
# If on 3.12, try: pip install mem0ai --no-build-isolation
```

### `sentence-transformers` / embedding errors during evaluate

The diversity metrics require `sentence-transformers` and its `all-MiniLM-L6-v2` model
(~90 MB, downloaded on first use). If it fails:
```bash
pip install sentence-transformers
python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('all-MiniLM-L6-v2')"
```

Or skip diversity metrics:
```bash
toolforge evaluate --in runs/dataset.jsonl --no-diversity --out reports/eval.json
```
